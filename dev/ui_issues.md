# Відомі проблеми UI (історія / нотатки)

Нижче — перелік проблем, які були зафіксовані, і що змінено в коді.

## 1. Зум і бокове меню

**Було:** при зменшенні масштабу сторінки (zoom) блоки контенту ніби «їхали» вправо до центру на Home, Dashboard, Profile, Settings.

**Зроблено:** відступ під фіксовану навігацію задається через `margin-left: var(--nav-width)` для `main.dashboard-container` замість `padding-left` на `body`, ширина бокової панелі збігається з `#nav` (одна CSS-змінна `--nav-width`). Основний контент вирівняно **ліворуч** у зоні поруч із меню, без `margin: 0 auto`, який на широких екранах візуально «тягнув» макет до центру. Додано `overflow-x: clip` для `html`/`body`, щоб прибирати зайвий горизонтальний скрол через округлення.

## 2. Екрани 4K і ширше

**Було:** на дуже великих дисплеях типографіка й відступи з `vw`/`vh` роздувалися, блоки здавалися зміщеними (у т.ч. «картинка»/фонова композиція — у правий кут), форми було незручно заповнювати.

**Зроблено:** для заголовка `#h1`, панелі welcome на Home, падінгів `.dashboard-container`, бейджа ролі та значень у налаштуваннях замінено занадто агресивні одиниці на `vmin`/`фіксовані rem` з обмеженням через `clamp`, щоб на 4K масштаб лишався передбачуваним. Для вузьких сторінок Profile/Settings додано клас `page-narrow` замість інлайнових стилів.

## 3. Інше

- У `nav.css` виправлено зламаний CSS-блок (стилі активного пункту меню перенесено на `.nav-link--active`).
- У шаблоні навігації додано підсвітку поточної сторінки через клас `nav-link--active`.

Якщо з’являться нові регресії на конкретних сторінках — дописуйте сюди коротко кроки відтворення і браузер.
